Input

Input let users enter and edit text.

Examples#

Basic Usage#

When you import <Input /> from mezzanine-ui, you will see:

<Input /><Input placeholder="Type something..." />

Sizes#

If you want to customize input size, we recommend using size prop.

<Input size="small" /><Input size="medium" /><Input size="large" />

Clearable Input#

In some cases, you might want to allow users to clear their typings via one button clicked instead of clicking backspace multiple times.

<Input  clearable  defaultValue="Clear Me"  placeholder="clearable"/>

Controlled vs Uncontrolled#

The component can be controlled and uncontrolled.

ControlledInput.tsx
const [value, setValue] = useState('Value');
return (  <Input    value={value}    onChange={(e) => setValue(e.target.value)}    placeholder="type something"  />);
UncontrolledInput.tsx
<Input  defaultValue="Default Value"  placeholder="type something"/>

Prefix & Suffix#

Sometimes you might want to have icons (or some decorators) for certain input to enhance the UX of the application as we recognize logos more easily than plain text. For example, if you have a search input you can add prefix with a search icon.

<Input  placeholder="Search"  prefix={<Icon icon={SearchIcon} />}/><Input  placeholder="Search"  suffix={<Icon icon={SearchIcon} />}/>

Also, you can render any decorator in prefix/suffix, but you need positioning your decorator by your own.
For example, if you use medium size input, then your decorator should remain height: 24px or line-height: 24px, which is equal to <Icon /> height, so decorator could be centerized.

Form States#

When user is typing, input state might changed several times due to your custom validation. <Input /> allows you to change different state via props such as disabled, error, readonly.

<Input disabled placeholder="disabled" /><Input error placeholder="error" /><Input readOnly placeholder="readonly" />

Integrate With FormField#

In most use cases, you will need to implement <Input /> with some form controller (such redux-form, formik, react-final-form...etc).
You can wrap <Input /> under <FormField /> which is also export from @mezzanine-ui/react to control form element states at once.

Note: To see how <FormField /> works, see FormField

<FormField required>  <Input placeholder="Required" /></FormField><FormField severity="error">  <Input placeholder="Error" /></FormField><FormField disabled>  <Input placeholder="Disabled" /></FormField>
